## 📐 框架间的联动规则

```
GOR方向说"重仓油气(25%)"
    → 个股四维用来选具体的油气标的
    → 对冲策略决定要不要买WTI $75 Put保护
    → 风控日历检查是否进入原油冲顶期（2026.08-09）

接盘论进入🟠晚期（7-9分）
    → GOR仓位自动打折（×0.7）
    → 对冲策略升级（从单个Put升级到Straddle）
    → 现金比例强制提高

Token美元稳定币立法通过（预计2026年内）
    → DXY修正因子的敏感度需要重新评估
    → 黄金底仓逻辑可能从15%上调
```

### 关键双向链路速查

| 链路 | 方向 | 触发场景 |
|------|:--:|------|
| GOR ↔ 对冲 | ⇄ | GOR极端区间→需对冲保护 / 对冲成本过高→重新评估GOR仓位 |
| GOR ↔ 风控 | ⇄ | GOR每日仓位 / 风控节点优先覆盖GOR方向 |
| GOR ↔ Token美元 | ⇄ | Token美元进度→DXY→GOR修正 / GOR修正异常→检查Token美元进度 |
| 接盘论 ↔ GOR | ⇄ | 接盘论阶段→GOR仓位打折 / GOR高仓位→确认接盘论未进入晚期 |
| 接盘论 ↔ 对冲 | ⇄ | 接盘论🟠→对冲升级为Straddle / 对冲触发→重新评估接盘论阶段 |
| 接盘论 ↔ 风控 | ⇄ | 接盘论预判市场阶段→风控日历给出时间锚点 |
| 对冲 ↔ 风控 | ⇄ | 风控节点→选择对应对冲策略 / 对冲到期→检查风控节点 |
| 审计 ↔ 全部 | ⇄ | 审计每月检查所有框架 / 所有框架的复盘都反馈到审计 |

## ⚠️ 规则冲突时的优先级

当多个框架给出不同指示时，按以下优先级执行。每个层级都有对应的深层链接：

```
1. 硬规则（不可被任何框架覆盖）
   ├── [[01-GOR方向框架#硬规则\|WTI<$75=减仓]]
   ├── [[01-GOR方向框架#硬规则\|不加杠杆]]
   └── [[01-GOR方向框架#硬规则\|不做空]]
         ↓
2. 风控日历节点规则（时间节点优先于方向判断）
   └── [[06-风控日历#风控日历的特殊地位\|风控优先级说明]]
         ↓
3. 接盘论防御降级（市场阶段优先于单个方向）
   └── [[03-接盘论框架#阶段判定\|接盘论阶段→仓位打折矩阵]]
         ↓
4. GOR方向框架（日常运行的默认规则）
   └── [[01-GOR方向框架#30秒速查\|GOR四区间仓位]]
         ↓
5. 个股四维研判（在方向框架允许的仓位内选标的）
   └── [[02-个股四维研判#决策矩阵\|四维决策矩阵]]
         ↓
6. 对冲策略选择（基于以上所有层级选择保护方案）
   └── [[05-对冲策略选择#30秒速查——对冲策略决策树\|对冲决策树]]
```
