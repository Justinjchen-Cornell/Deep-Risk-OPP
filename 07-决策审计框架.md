# 📊 决策一致性审计框架

> **一句话**：不是评判"你对不对"，而是区分"运气"和"能力"。
> **什么时候打开**：每月最后一个交易日收盘后。

---

## ⚡ 30秒速查

```
这个月你做了 N 个投资决策
  ├── 赚钱的决策 → 是因为判断对，还是因为运气好？
  ├── 亏钱的决策 → 是因为判断错，还是因为黑天鹅？
  └── 没做的决策 → 是错过了，还是正确放弃？

绩效归因 = 结果 - 运气 = 真正的能力
```

---

## 🎯 核心问题

机构投资者的分水岭不是"赚了多少钱"，而是**知道自己为什么赚、为什么亏**。

散户：这个月赚了=我很厉害 / 这个月亏了=市场不好
机构：这个月赚了=框架在X条件下有效 / 这个月亏了=框架在Y条件下失效

## 🔍 深度复盘问题（每月必答）

1. **本月最大的正确判断是什么？** 是基于哪个框架做出的？这个框架下个月还适用吗？

2. **本月最大的错误判断是什么？** 是框架本身的问题，还是执行的问题，还是纯粹的不可预测事件？

3. **有没有违反任何硬规则？**（加杠杆？追涨？不做对冲？WTI<$75还持有油气>5%？）

4. **有没有应该做但没做的交易？** 为什么没做——是忘了、是犹豫、还是正确判断不该做？

5. **下个月需要调整什么？** 框架参数？检查频率？执行纪律？

### 审计的元反馈循环

```
月度审计发现问题
    ↓
调整对应框架的参数（如GOR修正因子、四维维度权重、接盘论检查项）
    ↓
下月按新参数运行
    ↓
再次审计 → 验证调整是否改善了绩效
    ↓
12个月滚动 → 你知道自己的框架在什么市场环境下最有效
```

**这是整个决策体系的自我进化机制。** 没有审计，框架是死的。有了审计，框架和你一起进化。
